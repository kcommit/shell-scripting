# Deployment Audit Logs

This directory contains audit information generated by the deployment controller.

It starts without a deployment record. The final script creates or appends to:

```text
logs/deployment-audit.log
```

After running successful and failed workflows, display the records with:

```bash
cat logs/deployment-audit.log
```

Each record should include:

- Date and time
- Current user
- Application
- Environment
- Change ticket
- Artifact
- CPU usage
- Disk usage
- Final status

This README ensures that GitHub tracks the directory before the first workflow run.
